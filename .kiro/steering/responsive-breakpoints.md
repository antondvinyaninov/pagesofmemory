# Правило: Responsive Breakpoints

## Стандартные разрешения для проекта

В проекте используются **три основных breakpoint** для адаптивного дизайна:

### 1. Мобильные устройства (по умолчанию)
- **Разрешение**: до 640px
- **Префикс**: без префикса
- **Примеры**: `w-full`, `px-4`, `text-sm`

### 2. Планшеты
- **Разрешение**: 640px и выше
- **Префикс**: `sm:`
- **Примеры**: `sm:w-1/2`, `sm:px-6`, `sm:text-base`

### 3. Десктопы
- **Разрешение**: 1024px и выше
- **Префикс**: `lg:`
- **Примеры**: `lg:w-1/3`, `lg:px-8`, `lg:text-lg`

## Правила использования

1. **Мобильный first**: Всегда начинаем с мобильной версии (без префикса)
2. **Не используем** `md:` и `xl:` - только `sm:` и `lg:`
3. **Прогрессивное улучшение**: добавляем стили для больших экранов поверх мобильных

## Примеры

```html
<!-- Сетка: 1 колонка на мобильном, 2 на планшете, 3 на десктопе -->
<div class="grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4">

<!-- Отступы: маленькие на мобильном, средние на планшете, большие на десктопе -->
<div class="px-3 sm:px-4 lg:px-6">

<!-- Текст: маленький на мобильном, средний на планшете, большой на десктопе -->
<p class="text-sm sm:text-base lg:text-lg">

<!-- Скрыть на мобильном, показать на десктопе -->
<div class="hidden lg:block">
```

## Важно

- Всегда тестируем на всех трех разрешениях
- Используем только эти три breakpoint для консистентности
- Избегаем `md:` и `xl:` для упрощения кода
